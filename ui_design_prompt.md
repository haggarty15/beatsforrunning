# Beats For Running - UI/UX Design Prompt

This prompt is designed to be used with high-end image generators (like Midjourney or DALL-E) or as a creative brief for a frontend UI developer.

## The Concept

**Beats For Running** is a premium, high-energy web application for athletes. It algorithmically syncs a runner's music taste with their physical performance by mapping running pace to song BPM (Beats Per Minute).

---

## 🎨 Aesthetic & Visual Style

* **Vibe:** Athletic, futuristic, high-momentum, and premium.
* **Color Palette:** Carbon black background with vibrant, neon accents (Electric Orange `#FF5722`, Cyber Pink `#FF4081`, and Spotify Green `#1DB954`). Use deep gradients and glassmorphism.
* **Typography:** Modern, bold sans-serif (e.g., *Inter*, *Outfit*, or *Roboto Condensed*) to evoke speed and clarity.
* **Imagery:** Subtle, high-contrast silhouettes of runners or abstract light trails representing motion and sound waves.

---

## 📱 Key UI Components

1. **Dashboard Header:**
    * A sleek logo on the top-left using the electric orange theme.
    * Sart / Strave / Social navigation links with subtle glow-on-hover effects.
    * Spotify "Connected" status indicator in the top-right.

2. **Pace & Preference Input (The "Engine"):**
    * A centered, glassmorphic card.
    * **Pace Input:** Integration of a "clock-style" or "stepper" input for MM:SS format.
    * **Preference Sliders:** Toggle buttons for "High Cadence" or "Relaxed Power."
    * **Genre/Artist Pills:** Search bar that populates selected genres as glowing neon pills.

3. **The Animated BPM Gauge:**
    * As the user enters their pace, a dynamic circular gauge or sound-wave visualizer updates in real-time to show the "Target BPM Range" (e.g., 160-170 BPM).

4. **Playlist Reveal:**
    * A vertical list of curated tracks following the "Assemble" logic.
    * Each track card shows: Cover art (square), Song Title (Bold), Artist, and a small BPM tag.
    * A "Sync to Spotify" primary action button that pulses with a green glow.
